Introduction back to ToC

The HPC-FAIR Program part of the ontology provides classes and properties to semantically encode different program analyses, specifically relating to AST's and function callgraphs. AST nodes can be richly described using location information and analysis results provided by different graph links. This information, combined with other portions of the ontology, can help create a findable and fully described semantic description of a given program to provide knowledge graph insight into different types of software, such as machine learning models.

HPC Ontology for Programs: Description back to ToC

The HPC-FAIR Program part of the ontology provides mechanisms to semantically describe different program analyses, allowing for findability of different program constructs. Consider the following SAXPY code snippet ```c Z[i] = a * X[i] + Y[i]; ``` Assuming prior definitions of each variable are encoded into the ontology, an example encoding of this line can be represented as follows. ```rdf @prefix ex: <https://example.org/code#> @prefix hpc: <https://hpc-fair.github.io/ontology#> ex:SAXPY a hpc:AssignOp ; hpc:hasLeftOperand ex:Z_i ; hpc:hasRightOperand ex:add_mul ; hpc:columnNumber 14 . ex:Z_i a hpc:ArraySub ; hpc:hasLeftOperand ex:Z ; hpc:hasRightOperand ex:i ; hpc:columnNumber 10 . ex:add_mul a hpc:AddOp ; hpc:hasLeftOperand ex:a_mul_X ; hpc:hasRightOperand ex:Y_i ; hpc:columnNumber 25 . ex:a_mul_X a hpc:MulOp ; hpc:hasLeftOperand ex:a ; hpc:hasRightOperand ex:X_i ; hpc:columnNumber 18 . # ...continuing on to give information about the variables, X[i] ArraySub, and other metadata ```
