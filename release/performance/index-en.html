Abstract

The HPC-FAIR for Performance portion of the ontology provides data annotation properties relating to different aspects that affect the execution of a program. Timing, profiling, and diagnostic information are able to be encoded with these properties.

Introduction back to ToC

The goal for this portion of the ontology is to provide different annotation properties relating to performance. Aspects about the given hardware, operating system attributes, and program properties can be encoded using this ontology.

HPC Ontology for Performance: Description back to ToC

The HPC-FAIR Performance ontology contains only data properties, with the intended goal of annotating prior constructs (models, programs, etc.). Numerous properties are provides as to provide a snapshot of all aspects of a program's environment during execution, starting from hardware properties and going upwards to the program itself. For properties specifically relating to GPU performance, the [GPU](/gpu/index-en.html) ontology extension provides more in-depth data properties.
